      ETL Class

      The ETL class is the core component for managing all data loading operations from source files (e.g., CSV) into a destination MySQL database. It implements a robust, configuration-driven workflow based on a hierarchical structure of INI files:

      • Connectors.ini: The root file that lists all working directories (Connectors).
      • Master.ini: Within each Connector directory, this file defines shared settings and lists specific MetaData files.
      • MetaData.ini: Each MetaData file contains the precise instructions (table names, column types, etc.) for loading a single source data file.

      The class supports two primary modes of operation:

      • Scheduled Loading: Executed via the LoadAll() method, it automatically scans the file hierarchy. It checks if a load is due based on the configured schedule and, optionally, verifies the file's hash to ensure that only changed or new files are processed, minimizing redundant database operations.
      • On-Demand Loading: The LoadHttp() method allows for manual or external triggering (e.g., from a web service). This method bypasses the scheduled checks and forces the immediate loading of a specified file.

      The class also integrates with dedicated logging and ETL databases to track operation status, manage potential errors, and maintain a history of all loading activities.
